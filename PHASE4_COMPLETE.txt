ARCHITECTURE HIGHLIGHTS
================================================================================

QUEUE SYSTEM
├── postJobs Queue
│   ├── Capacity: Unlimited
│   ├── Workers: 5 concurrent (configurable)
│   └── Handler: handlePostAction()
├── commentJobs Queue
│   ├── Capacity: Unlimited
│   ├── Workers: 5 concurrent (configurable)
│   └── Handler: handleCommentAction()
├── reactionJobs Queue
│   ├── Capacity: Unlimited
│   ├── Workers: 5 concurrent (configurable)
│   └── Handler: handleReactionAction()
└── schedulerJobs Queue
    ├── Capacity: Unlimited
    ├── Workers: 1 (scheduler tasks)
    └── Handlers: handleSchedulerJob(), handleQuotaReset()

JOB FLOW
1. Job Created → Added to queue by scheduler/admin
2. Job Stored → Redis persists job data
3. Job Picked → Worker claims job based on availability
4. Pre-Check → Kill switch verified, account validated
5. Execution → Job handler processes the action
6. Logging → Action logged to Firestore
7. Metrics → Daily usage updated
8. Complete/Retry → Job marked complete or queued for retry

PRODUCTION DEPLOYMENT
Admin (Port 3000)    ─┐
  ↓                   ├──→ Nginx (Port 443 HTTPS)
Worker Process       ─┤
  ↓                   ├──→ Redis (Port 6379)
  └──────────────────→ Firestore (Cloud)

PM2 Management: Cluster mode × 2 instances per service
Auto-restart: On failure with exponential backoff
Memory limits: 500MB per instance
Log rotation: Automatic daily

  pnpm logs:worker        # Worker logs

BUILD FOR PRODUCTION
  pnpm build              # Build all
  pnpm build:admin        # Admin only
  pnpm build:worker       # Worker only

VIEW QUEUE STATUS
  curl http://localhost:3000/api/queue/status

ENABLE KILL SWITCH
  curl -X POST http://localhost:3000/api/policies/global \
    -H "Authorization: Bearer YOUR_TOKEN" \
    -d '{"globalKillSwitch": true}'

RETRY FAILED JOB
  curl -X POST http://localhost:3000/api/queue/jobs/{jobId}/retry \
    -H "Authorization: Bearer YOUR_TOKEN"

VIEW LOGS
  curl "http://localhost:3000/api/logs?status=failed&limit=50"
